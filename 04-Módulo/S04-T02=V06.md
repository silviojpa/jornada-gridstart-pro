## Tarefa 1 — função montar_mensagem_inicial()

* O ponto-chave é perceber que essa função segue exatamente o molde de formatar_status() do Bloco
2: parâmetros recebidos com local (usando nomes em minúsculo, diferentes das variáveis globais
URL, MAX_TENTATIVAS, INTERVALO — reforçando o que foi visto no Bloco 3 sobre escopo) e echo no
final para "devolver" a string montada, sem usar return.

* Erro comum esperado: usar os nomes URL, MAX_TENTATIVAS, INTERVALO (maiúsculo) como
nomes dos parâmetros local dentro da função. Funciona tecnicamente (porque local cria uma
cópia local com o mesmo nome), mas é uma inconsistência de estilo — as outras três funções do
script usam nomes em minúsculo para variáveis locais (url, max_tentativas, intervalo, tentativa,
mensagem, status). Vale comentar a convenção, mas não é um erro funcional.
* Erro comum esperado #2: usar return em vez de echo para "devolver" a string. return só aceita
números de 0 a 255 — não é possível devolver uma string formatada com return. Se o aluno tentar
isso, o script vai quebrar ou se comportar de forma inesperada (a string vira um código de erro
inválido).

## Tarefa 2 — captura e uso no fluxo principal

* Mesmo padrão de RESULTADO=$(checar_url "$URL" "$MAX_TENTATIVAS" "$INTERVALO"), já usado
mais abaixo no próprio script. A função é chamada com $(), o echo interno dela é capturado como
texto, e a variável resultante é exibida com echo no fluxo principal.
Solução alternativa válida: chamar echo "$(montar_mensagem_inicial "$URL"
"$MAX_TENTATIVAS" "$INTERVALO")" diretamente, sem variável intermediária. Funciona igual — a
variável MENSAGEM_INICIAL é só para clareza/legibilidade, não é obrigatória.
* Erro comum esperado: esquecer as aspas em echo "$MENSAGEM_INICIAL" (ou em
"$(montar_mensagem_inicial ...)"). Sem aspas, o Bash pode fazer word splitting na string — o
mesmo problema de $@ sem aspas, visto na V05. Como a mensagem aqui não tem múltiplos
espaços relevantes nem caracteres especiais, o efeito prático costuma ser pequeno, mas é um hábito
a reforçar.

## Tarefa 3 — comparação de saída (diff)

* Se a Tarefa 1 e a Tarefa 2 foram implementadas corretamente, diff não deve mostrar nenhuma
diferença — a string produzida por montar_mensagem_inicial() deve ser byte a byte igual
à linha original echo "=== Monitorando $URL (max: $MAX_TENTATIVAS tentativas, intervalo:
${INTERVALO}s) ===".
* Erro comum esperado: pequenas diferenças de formatação — por exemplo, espaço a mais/a menos
antes de tentativas, ou usar $INTERVALO em vez de ${INTERVALO}s (perdendo o s de "segundos"
no final). O diff vai apontar exatamente a linha divergente, o que ajuda a localizar o problema
rapidamente.
